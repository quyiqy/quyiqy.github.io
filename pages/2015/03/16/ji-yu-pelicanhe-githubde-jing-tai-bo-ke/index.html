基于Pelican和Github的静态博客

2015-03-16
By quyi

写在开头的话

首先感谢wwj大神对我的帮助，在他的帮助下，解决了我的许多疑惑。

让我们开始吧

你所看到的博客，是用Pelican搭建的。 Pelican是一套开源的使用Python编写的博客静态生成工具，它的特点如下： - 可以添加文章和和创建页面 - 可以使用MarkDown reStructuredText 和 AsiiDoc 的格式来书写 - 同时使用 Disqus评论系统, 支持 RSS和Atom输出, 插件, 主题, 代码高亮等功能 - 采用Jajin2模板引擎, 可以很容易的更改模板.

在搭建微博之前，建议先学习:git基本教程

起码你需要掌握以下知识

  1. 在github中创建一个repo：yourname.github.io
  2. git init #初始化git
  3. git add . #将工作区内容推送到暂存区
  4. git commit -m '你修改的内容' #提交暂存区内容
  5. git remote add origin http://github.com/yourname/yourname.github.io.git #将本地仓库与远程仓库链接起来
  6. git push -u origin master #将本地修改推送至远程

本文环境是ubuntu 14.04

安装Pelican

pip install pelican

安装markdown库，博主是利用vim作为编辑器，利用chrome的markdown preview plus扩展来写的

pip install Markdown

创建blog目录：

mkdir myblog
cd myblog

创建博客:

pelican-quickstart

问题都可以直接默认回车设置，不能直接默认的按照提示设置。之后可再myblog文件下修改pelicanconf.py

××至此，博客已经搭建好了××

写文章

××进入content目录，创建test.md××

cd content
vim test.md

博客内容必须有日期，标题

Date:2015-3-16
Title:test
Tags:pelican github #标签
category:test #分类
Author:quyi

××生成html×× 请务必再myblog文件路径下:

make html  #*在此感谢wwj大神的帮助*

博客撰写完毕，准备上传 进入output文件夹，发现多了一个test.html文件

cd output

××将博文推送到github××

git init
git add .
git commit -m 'first commit'
git remote add origin https://github.com/quyiqy/quyiqy.github.io.git #请将quyiqy替换为你的用户名
git push -u origin master

推送完毕，可以通过访问quyiqy.github.io来访问我的博客了。
